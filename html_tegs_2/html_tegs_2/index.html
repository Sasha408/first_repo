Глава 2 Керівні принципи для розробників Python

Наступні рекомендації охоплюють аспекти розробки MySQL програми, які можуть бути не відразу очевидними для розробників походить від фону Python

  • Для безпеки не закодуйте значення, необхідні для підключення, і Увійдіть в базу даних в своєму основному скрипті. Пітон має Конвенція амодуля, де ви може зберігати такі значення окремо від решти вашого коду.config.py
  • Скрипти Python часто створюють і знищують великі дані структури в пам'яті, аж до меж доступної оперативної пам'яті. Бо MySQL часто має справу з наборами даних, які в рази перевищують доступна пам'ять, методи, що оптимізують місце для зберігання і Вводу-виводу диска особливо важливі. Наприклад, в таблицях MySQL, зазвичай використовуються числові ідентифікатори, а не рядкові ключі словника, так що ключові значення компактні і мають передбачувана довжина. Особливо це важливо для колонок які складаютьпервинні ключдля придатності, тому що ті Значення стовпців дублюються в межах кожноговторинного індексу.InnoDB
  • Будь-яка програма, яка приймає введені дані, повинна очікувати, що вона погано впорається дані.

Погані дані можуть бути випадковими, наприклад, значення поза діапазоном або неправильно оформлені рядки. Додаток може використовувати перевірки на стороні сервера такі, якунікальні обмеженняіНЕ NULL обмеження, щоб погані дані ніколи не досягали база даних. На стороні клієнта використовуйте такі методи, як виняток обробники, щоб повідомити про будь-які проблеми та вжити коригувальних заходів.

Погані дані також можуть бути навмисними, що представляють собою атаку "SQL-ін'єкції". Наприклад, вхідні значення може містити лапки, крапку з комою, символи wildcard та інше символи, значущі в операторах SQL. Перевірка вхідних значень щоб переконатися, що у них є лише очікувані символи. Втеча від будь-якого спеціальні символи, які можуть змінити передбачувану поведінку, коли замінено на SQL-оператор. Ніколи не об'єднуйте користувача введення значення в оператор SQL без перевірки і втеча першою. Навіть при прийнятті вхідних даних, породжених деякими інша програма, очікуйте, що інша програма також може мати були скомпрометовані та надсилали вам неправильні або шкідливі дані.%_

  • Оскільки набори результатів із запитів SQL можуть бути дуже великими, використовуйте Відповідний метод отримання елементів із набору результатів як ви петляєте через них. fetchone()отримує один елемент, коли ви знаєте, що набір результатів містить однорядний. fetchall()отримує всі елементи, коли ви знаєте, що набір результатів містить a обмежена кількість рядків, які можуть зручно вписатися в пам'ять. fetchmany() - це метод загального призначення, коли ви не можете передбачити розмір набору результатів: ви продовжуєте викликати його та зациклюватися на повернуті елементи, поки більше не буде результатів для обробки.
